# Darask Paint

超高速起動・軽量・シンプルな Windows 用ラスタ画像編集ソフトです。
Krita のような多機能ペイントソフトの起動待ちにうんざりした人向けに、
「起動が速いこと」を最優先に作られています。

Rust + eframe/egui 製、単一 exe(ランタイム・インストーラ不要)。UI はすべて日本語です。

スクリーンショットはありません(README では省略しています)。

## 特徴

- **起動が速い**: プロセス起動から最初のフレーム描画まで実測 **約 190〜220ms**(目標 300ms 以内)。
- **軽い**: 配布物は単一 exe、実測 **約 7.0MB**(目標 15MB 以下)。ランタイムのインストール不要。
- **アイドル時は CPU をほぼ使わない**: 入力がない限り再描画しません。
- ひととおりの基本ツール(ペン・消しゴム・直線・矩形・楕円・塗りつぶし・スポイト・選択・手のひら)、
  アンドゥ/リドゥ、PNG/JPEG/BMP/GIF/WebP の読み込みと PNG/JPEG/BMP の保存、
  クリップボード連携、ドラッグ&ドロップに対応しています。
- **レイヤー**(最大 64 枚、通常合成・表示切替・不透明度)に対応。保存時は常に可視レイヤーを統合して書き出します。
- 右パネルの**色相リング + SV 三角形のカラーパネル**でプライマリ/セカンダリ色を選択できます
  (定義済みパレット・ユーザーパレット・HEX 入力・最近使った色つき)。
- 左ツールバーのアイコンはすべて `egui::Painter` によるベクター描画(画像アセット・絵文字フォント不使用)。
- 選択・浮動片の外周に **8 個のスケールハンドル**が表示され、ドラッグで拡縮できます(Shift で縦横比固定)。
- 筆圧・テキストツール・フィルタ・印刷・プラグイン・多言語 UI・設定の永続化・レイヤー付きネイティブ保存形式は
  対象外です(詳細は `docs/SPEC.md` の「非目標」を参照)。

## 使い方

1. `darask-paint.exe` を起動すると、1280×720・白背景の新規ドキュメントが自動的に開きます
   (メモ帳や MS ペイントと同じ感覚で、すぐに描き始められます)。
2. 画像ファイルをエクスプローラーからウィンドウにドラッグ&ドロップするか、
   「ファイル → 開く」(`Ctrl+O`)で画像を開けます。
   また `darask-paint.exe 画像.png` のようにコマンドライン引数でファイルを直接開くこともできます
   (「プログラムから開く」に対応)。
3. 左のツールバーからツールを選び、キャンバス上でドラッグして描画します。
   オプションバーでブラシサイズ・ツール固有のオプション(塗りつぶし許容値、図形の塗り方など)を調整できます。
   色は右パネルのカラーパネルで選びます。
4. 右パネルのレイヤーセクションでレイヤーの追加・複製・削除・並べ替え・表示切替・不透明度・下と結合・統合ができます。
5. `Ctrl+S` で上書き保存、`Ctrl+Shift+S` で名前を付けて保存します。複数レイヤーがある状態で保存すると、
   統合して書き出したことをトーストで通知します。未保存の変更がある状態で閉じる・新規作成・開くを行おうとすると、
   保存確認のダイアログが出ます。

透明ピクセルの下には市松模様が表示されます。ズームは `Ctrl+ホイール`(カーソル位置を中心に段階的に拡大縮小)、
パンはホイール(縦)・`Shift+ホイール`(横)・`Space` を押しながらのドラッグ・中ボタンドラッグで行えます。

## レイヤー

- ドキュメントは 1 枚以上のレイヤーを持ち、下から上へ通常合成します(ブレンドモードなし)。上限 64 枚。
- 描画系ツール(ペン・消しゴム・図形・塗りつぶし)、選択・浮動片、クリップボード操作は**アクティブレイヤーのみ**に作用します。
  スポイトは合成結果から色を取ります。
- 右パネルのレイヤー一覧: 行 = 表示チェック + 名前。クリックでアクティブ化、ダブルクリックで名前変更。
  ボタンは 新規 / 複製 / 削除 / 上へ / 下へ / 下と結合。レイヤーが 1 枚のときは削除・結合は無効です。
- アクティブレイヤーの不透明度スライダーがあります。
- メニュー「レイヤー」: 新規レイヤー (`Ctrl+Shift+N`) / レイヤーを複製 / レイヤーを削除 / 上へ移動 / 下へ移動 /
  下のレイヤーと結合 (`Ctrl+E`) / 画像の統合 (`Ctrl+Shift+E`)。
- 表示切替と不透明度変更は undo 履歴に積まれません。それ以外のレイヤー操作(追加/複製/削除/移動/結合/統合)は
  1 回で 1 つの undo 単位になります。

## カラーパネル

右パネル上段に常時表示され、egui 標準のポップアップピッカーは使いません。

- プライマリ/セカンダリの重ね表示スウォッチ + 入替ボタン(`X` キーと同じ)。
- 色相リング(外周ドーナツ、0°=赤を 12 時方向・時計回り)+ 内接する固定向きの正三角形(彩度・明度)。
- アルファスライダー(市松模様の下地付き)と HEX 入力欄(`#RRGGBB` / `#RRGGBBAA`)。
- 定義済みパレット 24 色(2 段)。クリックでプライマリ、右クリックでセカンダリに設定。「＋」でプライマリ色を
  ユーザーパレットに追加、ユーザー色は右クリックメニューで削除できます(永続化はしません)。
- 最近使った色 8 個(描画確定時に使用色が先頭に追加されます)。

## 選択・浮動片のスケールハンドル

- 選択矩形および浮動片の外周に、四隅+各辺中央の 8 個のハンドル(約 7pt 角)が表示されます。
- ハンドルをドラッグすると拡縮できます(四隅=縦横同時、辺=単軸)。`Shift` で縦横比を固定したまま拡縮します。
- まだ浮動化していない選択のハンドルを掴んだ場合は、内部ドラッグと同じく自動的に浮動化してから拡縮します。
- 拡縮のたびに浮動化した時点の元ピクセルからバイリニアで再サンプリングするため、繰り返し拡縮しても劣化しません
  (最小 1px、最大 8192px)。
- ハンドルにカーソルを重ねるとリサイズカーソルに変わります。

## ショートカット一覧

### ツール(左ツールバー)

| ツール | キー | 備考 |
|---|---|---|
| ペン | `B` | 左ドラッグ=プライマリ色、右ドラッグ=セカンダリ色。オプションでアンチエイリアス切替可 |
| 消しゴム | `E` | 丸筆ハードエッジで透明化 |
| 直線 | `L` | `Shift` で 45° 単位に拘束 |
| 矩形 | `R` | `Shift` で正方形に拘束 |
| 楕円 | `C` | `Shift` で正円に拘束 |
| 塗りつぶし | `F` | スキャンライン flood fill。オプションバーで許容値を調整 |
| スポイト | `I` | 左クリック=プライマリ、右クリック=セカンダリに取得 |
| 選択 | `M` | フローティング方式・スケールハンドル付き(下記参照) |
| 手のひら | `H` | ドラッグでパン |
| ブラシサイズ増減 | `[` / `]` | 1〜64px |
| 一時スポイト | `Alt` + クリック | 描画系ツール使用中でも一時的にスポイトに切り替え |

### 色

| 操作 | キー |
|---|---|
| プライマリ/セカンダリ入れ替え | `X` |
| 最近使った色(最大8個) | クリックでプライマリに設定 |
| パレット | クリック=プライマリ、右クリック=セカンダリ |

### 選択・フローティング

| 操作 | キー |
|---|---|
| 全選択 | `Ctrl+A` |
| 選択解除 | `Ctrl+D` |
| コピー | `Ctrl+C` |
| 切り取り | `Ctrl+X` |
| 貼り付け | `Ctrl+V`(未編集の白紙ドキュメントならその画像サイズに置き換え) |
| 削除(透明化) | `Delete` |
| 拡縮 | ハンドルをドラッグ(`Shift` で縦横比固定) |
| 浮動片を確定 | `Enter` / 選択外クリック / ツール切替 |
| その場で確定して選択解除 | `Esc` |

### レイヤー

| 操作 | キー |
|---|---|
| 新規レイヤー | `Ctrl+Shift+N` |
| 下のレイヤーと結合 | `Ctrl+E` |
| 画像の統合 | `Ctrl+Shift+E` |

複製・削除・上下移動には専用ショートカットはなく、右パネルのボタンかメニューから操作します。

### ファイル

| 操作 | キー |
|---|---|
| 新規 | `Ctrl+N` |
| 開く | `Ctrl+O` |
| 上書き保存 | `Ctrl+S` |
| 名前を付けて保存 | `Ctrl+Shift+S` |
| 終了 | `Alt+F4` |

### 編集

| 操作 | キー |
|---|---|
| 元に戻す | `Ctrl+Z` |
| やり直し | `Ctrl+Y` または `Ctrl+Shift+Z` |

「画像」メニュー(画像サイズ変更・キャンバスサイズ変更・選択範囲でトリミング・反転・回転)にはショートカットはなく、メニューから操作します(いずれも全レイヤーに適用され、1 回で 1 つの undo 単位になります)。

- **画像サイズ変更…**: 幅・高さを指定して拡大縮小します。「縦横比を固定」チェック(デフォルト ON)で片方を変えるともう片方が自動追従します。補間方法はバイリニア/ニアレストから選べます。
- **キャンバスサイズ変更…**: 幅・高さを指定してキャンバス(作業領域)自体を拡張・縮小します。既存の画像は左上基準で配置され、拡張された部分は全レイヤーとも透明になります。
- **選択範囲でトリミング**: アクティブな選択範囲(または浮動片)に合わせてキャンバスを切り抜きます。選択がない場合は何もしません。

## ビルド方法

前提: Rust stable(x86_64-pc-windows-msvc)がインストールされていること。

```powershell
# 開発ビルド
cargo build

# テスト
cargo test

# 配布用リリースビルド(単一 exe を target/release/darask-paint.exe に生成)
cargo build --release
```

依存 crate は `eframe` / `image` / `rfd` / `arboard` の4つのみです(`Cargo.toml` 参照)。

### 起動ベンチマーク(スモークテスト)

GUI を対話的に操作せずに起動時間を計測できます。環境変数 `DARASK_BENCH=1` を設定して実行すると、
2 フレーム目描画後にカレントディレクトリの `bench.txt` に起動ミリ秒を書き出して自動終了します。

```powershell
$env:DARASK_BENCH = '1'
& .\target\release\darask-paint.exe
Get-Content .\bench.txt
```

手元の実測値: **約 190〜220ms**(目標 300ms 以内)、exe サイズ **約 7.0MB**(目標 15MB 以下)。

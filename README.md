# Darask Paint

[![CI](https://github.com/daraskme/darask-paint/actions/workflows/ci.yml/badge.svg)](https://github.com/daraskme/darask-paint/actions/workflows/ci.yml)

超高速起動・軽量・シンプルな Windows 用ラスタ画像編集ソフトです。
Krita のような多機能ペイントソフトの起動待ちにうんざりした人向けに、
「起動が速いこと」を最優先に作られています。

Rust + eframe/egui 製、単一 exe(ランタイム・インストーラ不要)。UI はすべて日本語です。

スクリーンショットはありません(README では省略しています)。

## 特徴

- **起動が速い**: プロセス起動から最初のフレーム描画まで実測 **約 190〜220ms**(目標 300ms 以内)。
- **軽い**: 配布物は単一 exe、実測 **約 7.3MB**(目標 15MB 以下)。ランタイムのインストール不要。
- **アイドル時は CPU をほぼ使わない**: 入力がない限り再描画しません。
- ひととおりの基本ツール(ブラシ・消しゴム・直線・矩形・楕円・塗りつぶし・グラデーション・スポイト・
  矩形選択・楕円選択・なげなわ・自動選択・移動・テキスト・手のひら・ズーム)、
  全件アンドゥ/リドゥ、履歴とレイヤーを保存する `.dpaint`、PNG/JPEG/BMP/GIF/WebP の読み込みと PNG/JPEG/BMP の画像書き出し、
  クリップボード連携、ドラッグ&ドロップに対応しています。
- **タブで複数ドキュメントを同時に開けます**(最大 24 枚)。`Ctrl+N`/`Ctrl+O` は新規タブを追加して開き、
  既に開いているファイルを開こうとした場合はそのタブに切り替わります。選択・浮動片・アンドゥ履歴・
  ズーム/パンはタブごとに独立し、ツール・色・ブラシ設定はアプリ全体で共有されます(詳細は下記「タブ」節)。
- **グラデーションツール**(`G`、`Shift+G` で塗りつぶしと巡回): ドラッグで始点→終点を指定し、
  線形/円形・プライマリ→セカンダリ/プライマリ→透明をオプションバーで選べます。
  ドラッグ中はキャンバス自体にライブプレビューが表示され、選択範囲があればその中だけに効きます。
- **色調補正**(「画像」メニュー): 明るさ・コントラスト…・色相/彩度/明度…(`Ctrl+U`)は
  ライブプレビュー付きモーダル、階調の反転(`Ctrl+I`)・グレースケール化(`Ctrl+Shift+U`)は即時適用です。
  いずれもアクティブレイヤー対象、選択範囲があればその中だけに効き、1 回の適用が 1 つの undo 単位になります。
- **ブラシのスムージング(手ブレ補正)**: ブラシ/消しゴム/鉛筆モード共通のオプション(0〜100%)。
  入力座標を指数移動平均で滑らかにし、ストローク終端では実際のポインタ位置まで追いつきます。
- **ピクセルグリッド**(「表示」メニューでトグル、デフォルト ON): ズーム 800% 以上で画像ピクセルの境界に
  薄いグリッド線を表示します。
- **選択はビットマスク方式**: 矩形に加えて楕円選択・なげなわ(自由/多角形)・自動選択(マジックワンド)で
  任意形状の選択ができます。選択・浮動化・スケールハンドル拡縮・コピー/切り取り/貼り付け・削除・自由変形は
  すべてこのマスク形状のまま動作し、描画系ツール(ブラシ・消しゴム・図形・塗りつぶし)は選択範囲内にのみ効きます。
- **テキストツール**(`T`): クリックした位置にインラインの複数行テキストボックス(IME 対応)を表示します。
  `Ctrl+Enter` またはボックス外クリックで確定するとアンチエイリアス付きでラスタライズされ、
  選択/移動ツールと同じ浮動片(移動・スケールハンドル拡縮・`Enter`確定・`Esc`破棄)になります。
  色はプライマリ色、フォントはオプションバーで 8〜144px に調整でき、UI と同じシステム日本語フォントを使います。
- **移動ツール**: ドラッグでアクティブレイヤー全体(選択範囲があればその範囲だけ)を移動できます。
  内部的には選択ツールと同じ浮動化・スケールハンドル機構を使います。
- **自由変形**(`Ctrl+T`): 選択範囲(無ければ全体)を浮動化してスケールハンドルで拡縮できます。
- **ズームツール**: クリックでクリック位置を中心に 1 段階拡大、`Alt+クリック` で縮小します。
- **ブラシ/消しゴムの共通ストロークエンジン**: 硬さ(エッジのなめらかさ)・不透明度(1 ストローク内は上限を超えず、
  ストロークをまたぐと重なる PS 準拠の意味論)・鉛筆モード(ピクセルアート向けの2値スタンプ)・
  `Shift` クリックでの直線連結に対応しています。
- **レイヤー**(最大 64 枚、通常合成・表示切替・不透明度)に対応。`.dpaint` はレイヤーとundo/redo全件を保持し、PNG/JPEG/BMPは可視レイヤーを統合して書き出します。
- 右パネルの**色相リング + SV 三角形のカラーパネル**でプライマリ/セカンダリ色を選択できます
  (定義済みパレット・ユーザーパレット・HEX 入力・最近使った色つき)。
- 左ツールバーおよびメニューバーは `egui::Painter` のベクターアイコン上、短い日本語ラベル下の一体タイル(画像アセット・絵文字フォント不使用)。
  メニューバーは開閉するドロップダウンではなく、**常時表示の操作タイル行**です(ウィンドウ幅が足りない場合は
  次の行へ折り返します)。「最近使ったファイル」だけはクリックで小さなポップアップリストを開きます。
  各アイコンの実際の意味・確認ダイアログ・ショートカットは以前のドロップダウンメニューと同じです。
- 選択・浮動片の外周に **8 個のスケールハンドル**が表示され、ドラッグで拡縮できます(Shift で縦横比固定)。
- **設定の永続化**: ウィンドウ寸法・最大化状態、ブラシ/ツールのオプション、プライマリ/セカンダリ色、
  ユーザーパレット、最後に使ったツール、ピクセルグリッド表示、最近使ったファイル(最大 8)を
  `%APPDATA%\darask-paint\settings.txt` に保存し、次回起動時に復元します(詳細は下記「設定の永続化」節)。
- 筆圧・フィルタ・印刷・プラグイン・多言語 UI・
  太字/斜体/フォント選択(テキストツールはシステム日本語フォント1種のみ)は
  対象外です(詳細は `docs/SPEC.md` の「非目標」を参照)。

## インストール

インストーラは不要です。以下のどちらかの方法で入手できます。

1. **リリース版をダウンロード**(推奨): [Releases](https://github.com/daraskme/darask-paint/releases)
   ページから最新の `darask-paint-vX.Y.Z-windows-x64.zip` をダウンロードして展開し、
   中の `darask-paint.exe` を実行するだけです。`v*` タグの push ごとに GitHub Actions が
   自動でビルド・zip 化・リリース作成まで行います(`.github/workflows/release.yml`)。
2. **自分でビルド**: 下記「ビルド方法」を参照してください。

## 使い方

1. `darask-paint.exe` を起動すると、1280×720・白背景の新規ドキュメントが自動的に開きます
   (メモ帳や MS ペイントと同じ感覚で、すぐに描き始められます)。
2. 画像ファイルをエクスプローラーからウィンドウにドラッグ&ドロップするか、
   「ファイル → 開く」(`Ctrl+O`)で画像を開けます(複数ファイルを同時にドロップすると、それぞれ別タブで開きます)。
   また `darask-paint.exe 画像.png` のようにコマンドライン引数でファイルを直接開くこともできます
   (「プログラムから開く」に対応)。「ファイル → 新規」(`Ctrl+N`)・「開く」はどちらも新しいタブを追加して
   開きます(既存タブの内容は変わりません)。開こうとしたファイルが既に別タブで開いていれば、
   新規タブを作らずそのタブに切り替わります。
3. 左のツールバーからツールを選び、キャンバス上でドラッグして描画します。
   オプションバーでブラシサイズ・ツール固有のオプション(塗りつぶし許容値、図形の塗り方など)を調整できます。
   色は右パネルのカラーパネルで選びます。
4. 右パネルのレイヤーセクションでレイヤーの追加・複製・削除・並べ替え・表示切替・不透明度・下と結合・統合ができます。
5. `Ctrl+S` で上書き保存、`Ctrl+Shift+S` で名前を付けて保存します。無題の既定は `.dpaint` です。画像形式を選んだ場合だけ複数レイヤーを
   統合して書き出したことをトーストで通知します。未保存の変更がある状態で閉じる・新規作成・開くを行おうとすると、
   保存確認のダイアログが出ます。

透明ピクセルの下には市松模様が表示されます。ズームは `Ctrl+ホイール`(カーソル位置を中心に段階的に拡大縮小)、
パンはホイール(縦)・`Shift+ホイール`(横)・`Space` を押しながらのドラッグ・中ボタンドラッグで行えます。

## タブ(複数ドキュメント)

- メニューバーの直下にタブバーがあり、複数のドキュメントを同時に開けます(上限 24 枚。超えて開こうとすると
  作成せずトースト通知します)。各タブにはファイル名(無題なら「無題」「無題2」…と連番)・未保存インジケータ
  (`*`)・閉じるボタン(`×`)が表示され、クリックでアクティブ化、**中クリックでそのタブを閉じます**。
- **常に 1 タブ以上を維持**します。最後の 1 タブを閉じようとした場合は、閉じる代わりに「新規」と同じ扱い
  (未保存なら確認してから内容を白紙に戻す)になります。
- `Ctrl+N`(新規)・`Ctrl+O`(開く)はアクティブタブの内容を置き換えず、**新規タブを追加してそこに開きます**。
  開こうとしたファイルが既に開いているタブがあれば、新規タブを作らずそのタブへ切り替わります。
  ドラッグ&ドロップ・「最近使ったファイル」から開く場合も同じ規則です。
- `Ctrl+W` でアクティブタブを閉じます(未保存なら確認ダイアログ)。`Ctrl+Tab` / `Ctrl+Shift+Tab` で
  次/前のタブへ切り替わります(端では反対側へ循環)。ウィンドウを閉じる操作は、未保存のタブがあれば
  タブごとに順番に確認します。
- タブごとに独立するもの: 選択・浮動片・アンドゥ履歴・ズーム/パン・ファイルパス・未保存フラグ。
  アプリ全体で共有するもの: ツール・色・ブラシ設定・パレット・最近使ったファイル・ウィンドウ状態。
- タブの並べ替え(ドラッグでの順序変更)やセッション復元(前回開いていたタブ一覧の記憶)は非対応です。

## レイヤー

- ドキュメントは 1 枚以上のレイヤーを持ち、下から上へ通常合成します(ブレンドモードなし)。上限 64 枚。
- 描画系ツール(ペン・消しゴム・図形・塗りつぶし)、選択・浮動片、クリップボード操作は**アクティブレイヤーのみ**に作用します。
  スポイトは合成結果から色を取ります。
- 右パネルのレイヤー一覧: 行 = 表示チェック + 名前。クリックでアクティブ化、ダブルクリックで名前変更。
  ボタンは 新規 / 複製 / 削除 / 上へ / 下へ / 下と結合。レイヤーが 1 枚のときは削除・結合は無効です。
- アクティブレイヤーの不透明度スライダーがあります。
- メニュー「レイヤー」: 新規レイヤー (`Ctrl+Shift+N`) / レイヤーを複製 (`Ctrl+J`) / レイヤーを削除 / 上へ移動 / 下へ移動 /
  下のレイヤーと結合 (`Ctrl+E`) / 画像の統合 (`Ctrl+Shift+E`)。
- 表示切替と不透明度変更は undo 履歴に積まれません。それ以外のレイヤー操作(追加/複製/削除/移動/結合/統合)は
  1 回で 1 つの undo 単位になります。

## カラーパネル

右パネル上段に常時表示され、egui 標準のポップアップピッカーは使いません。

- プライマリ/セカンダリの重ね表示スウォッチ + 入替ボタン(`X` キーと同じ)。
- 色相リング(外周ドーナツ、0°=赤を 12 時方向・時計回り)+ 内接する固定向きの正三角形(彩度・明度)。
- アルファスライダー(市松模様の下地付き)と HEX 入力欄(`#RRGGBB` / `#RRGGBBAA`)。
- 定義済みパレット 24 色(2 段)。クリックでプライマリ、右クリックでセカンダリに設定。「＋」でプライマリ色を
  ユーザーパレットに追加、ユーザー色は右クリックメニューで削除できます(ユーザーパレットは設定ファイルに
  永続化されます)。
- 最近使った色 8 個(描画確定時に使用色が先頭に追加されます。こちらは永続化されません)。

## 選択ツールの拡充(楕円・なげなわ・自動選択)

- **矩形選択 / 楕円選択**(`M` / `Shift+M` で巡回): `Shift` を押しながらドラッグすると正方形/正円に拘束されます。
- **なげなわ**(`L`、`Shift+L` で自由↔多角形モードを切替):
  - 自由モードはドラッグの軌跡をそのまま閉じてマスク化します。
  - 多角形モードはクリックで頂点を追加し、始点付近をクリック・ダブルクリック・`Enter` のいずれかで閉じます。
    `Esc` で作成中の多角形を中止できます(選択自体には影響しません)。
- **自動選択**(`W`): クリックした画素から許容値(オプションバー、0〜255)内の連結領域を選択します。
  塗りつぶしと同じ判定で、アクティブレイヤーの画素を基準にします。
- いずれのツールも、確定済みの選択がある状態で新規に選択すると**置き換え**られます(加算/減算は非対応)。
  選択後は移動ツール(`V`)や矩形選択ツール(`M`)でそのまま移動・スケールハンドル拡縮ができます
  (どのツールで作った選択でも共通の浮動化・ハンドル機構に乗ります)。

## 選択・浮動片のスケールハンドル

- 選択(矩形・楕円・なげなわ・自動選択のいずれの形状でも)および浮動片の外周に、
  四隅+各辺中央の 8 個のハンドル(約 7pt 角)が表示されます。
- ハンドルをドラッグすると拡縮できます(四隅=縦横同時、辺=単軸)。`Shift` で縦横比を固定したまま拡縮します。
- まだ浮動化していない選択のハンドルを掴んだ場合は、内部ドラッグと同じく自動的に浮動化してから拡縮します。
- 拡縮のたびに浮動化した時点の元ピクセルからバイリニアで再サンプリングするため、繰り返し拡縮しても劣化しません
  (最小 1px、最大 8192px)。
- ハンドルにカーソルを重ねるとリサイズカーソルに変わります。

## 設定の永続化

- 保存先は `%APPDATA%\darask-paint\settings.txt`(手書きの `キー\t値` 形式の UTF-8 テキスト、外部
  crate は使わず自前のパーサ/シリアライザで読み書きします)。
- 保存される項目: ウィンドウ寸法・最大化状態 / 最近使ったファイル(最大 8) / ブラシ設定(サイズ・硬さ・
  不透明度・鉛筆モード・スムージング) / ツールオプション(塗りつぶし・自動選択の許容値、図形の塗り方、
  グラデーションの種類・色) / プライマリ・セカンダリ色 / ユーザーパレット / 最後に使ったツール /
  ピクセルグリッド表示 / **履歴パネルの表示件数**。
- 読み込みは起動時に 1 回だけ、書き込みは終了時と「最近使ったファイル」更新時・設定ダイアログの OK 時に行います。
- 設定ファイルが存在しない・壊れている場合は、黙って既定値にフォールバックします(起動が失敗することは
  ありません)。
- **ファイル → 最近使ったファイル**: 最近開いた/保存したファイルへのショートカットです。存在しなくなった
  ファイルを選ぶと、一覧から取り除いた上でトーストで通知します。
- **ヘルプ → バージョン情報**: バージョン番号とリポジトリ URL を表示する小さなモーダルです。

## 設定(環境設定)ダイアログ・アンドゥ履歴パネル

- **設定ダイアログ**(`Ctrl+K`、またはツールバー/メニューバーの歯車アイコン): 「履歴パネルの表示件数」
  (1〜500、デフォルト 50)を数値入力で変更できます。OK で即座に適用+設定ファイルへ保存され、
  **開いている全タブ**に反映されます。値を下げてもundo/redo自体は削除されず、`.dpaint`には全件保存されます。
  キャンセルすると変更は破棄されます。
- **アンドゥ履歴パネル**(右パネルの「履歴」セクション、アクティブタブの履歴を表示・タブ切替に追随):
  現在位置前後を時系列で表示し、各行は操作の日本語ラベル(例:「ブラシ」「塗りつぶし」
  「画像サイズ変更」)です。現在位置がハイライトされ、それより後ろ(やり直し可能な範囲)は淡色表示になります。
  **行をクリックするとその時点まで一気に戻る/進みます**(進行中のストローク・浮動片があれば、既存の
  元に戻す/やり直しと同じ安全規則で先に確定してからジャンプします)。ジャンプ後に新しい操作を行うと、
  淡色表示だった「やり直し」範囲は通常どおり破棄されます。表示件数より古いエントリも保持され、一覧には
  「(古い履歴もプロジェクトに保存されます)」と表示されます。ツールバー/メニューバーの元に戻す・やり直しは従来どおり単発ステップの操作のままです
  (複数ステップのジャンプは履歴パネルのクリックでのみ行います)。

## ショートカット一覧

Photoshop 準拠のキーマップです(SPEC §20。旧 v1/v2 の `L`/`R`/`C`/`F` は廃止・変更されました。
v4 で `L` はなげなわとして復活し、`G` は塗りつぶし/グラデーションの巡回(`Shift+G`)にも使われます)。
実装上もこのキーマップは `src/keymap.rs` の `KEYMAP` という単一のテーブルが唯一の情報源で、
メニュー・ツールチップの表記もすべてそこから生成されるため、下表と実際の挙動がずれることはありません。

### ツール(左ツールバー)

| ツール | キー | 備考 |
|---|---|---|
| 移動 | `V` | ドラッグでアクティブレイヤー全体(選択範囲があればその範囲だけ)を移動。選択と同じ浮動化・ハンドル機構を共有 |
| 矩形選択 / 楕円選択 | `M` / `Shift+M` | `M` で直前に使った形状(矩形/楕円)を選択、`Shift+M` で巡回。フローティング方式・スケールハンドル付き(下記参照)。`Shift` ドラッグで正方形/正円に拘束 |
| なげなわ | `L` / `Shift+L` | `L` でなげなわを選択、`Shift+L` で自由↔多角形モードを切替。自由=ドラッグの軌跡を閉じてマスク化、多角形=クリックで頂点追加・始点クリック/ダブルクリック/`Enter`で閉じる・`Esc`で中止 |
| 自動選択 | `W` | クリック画素から許容値(オプションバー)内の連結領域を選択(塗りつぶしと同じ判定、アクティブレイヤー基準) |
| ブラシ(旧ペン) | `B` | 丸筆・常時アンチエイリアス。左ドラッグ=プライマリ色、右ドラッグ=セカンダリ色。硬さ・不透明度・鉛筆モード・スムージング(手ブレ補正)はオプションバーで調整 |
| 消しゴム | `E` | ブラシと同じエンジン。不透明度は「強さ」として働く(カバレッジ×強さぶんアルファを減らす) |
| 塗りつぶし / グラデーション | `G` / `Shift+G` | `G` で直前に使った方(塗りつぶし/グラデーション)を選択、`Shift+G` で巡回。塗りつぶしはスキャンライン flood fill(許容値をオプションバーで調整)。グラデーションはドラッグで始点→終点、種類(線形/円形)・色(プライマリ→セカンダリ/プライマリ→透明)をオプションバーで選ぶ |
| スポイト | `I` | 左クリック=プライマリ、右クリック=セカンダリに取得 |
| テキスト | `T` | クリックでインライン編集ボックス(複数行・IME対応)を表示。`Ctrl+Enter`/ボックス外クリックで確定し浮動片になる。`Esc` で入力破棄 |
| 図形(直線/矩形/楕円) | `U` / `Shift+U` | `U` で直前に使った図形ツールを選択、`Shift+U` で 直線→矩形→楕円 を巡回(Photoshop のツールグループと同じ考え方。旧 `L`/`R`/`C` は廃止)。各図形は `Shift` で拘束(直線=45°単位、矩形=正方形、楕円=正円) |
| 手のひら | `H` | ドラッグでパン |
| ズーム | `Z` | クリックで 1 段階拡大、`Alt+クリック` で縮小 |
| ブラシサイズ増減 | `[` / `]` | 1〜64px |
| ブラシ硬さ増減 | `Shift+[` / `Shift+]` | ブラシ/消しゴム共通、±10% |
| ブラシ不透明度(強さ) | `1`〜`9` / `0` | 10%〜90% / 100%(ブラシ/消しゴム共通) |
| ブラシで直線連結 | `Shift` + クリック | 直前のストローク終点からクリック位置まで直線を引く |
| 一時スポイト | `Alt` + クリック | 描画系ツール使用中でも一時的にスポイトに切り替え |

### 編集

| 操作 | キー |
|---|---|
| 元に戻す | `Ctrl+Z` |
| やり直し | `Ctrl+Y` または `Ctrl+Shift+Z` |

「画像」メニュー(画像サイズ変更・キャンバスサイズ変更・選択範囲でトリミング・反転・回転)にはショートカットはなく、メニューから操作します(いずれも全レイヤーに適用され、1 回で 1 つの undo 単位になります)。

- **画像サイズ変更…**: 幅・高さを指定して拡大縮小します。「縦横比を固定」チェック(デフォルト ON)で片方を変えるともう片方が自動追従します。補間方法はバイリニア/ニアレストから選べます。
- **キャンバスサイズ変更…**: 幅・高さを指定してキャンバス(作業領域)自体を拡張・縮小します。既存の画像は左上基準で配置され、拡張された部分は全レイヤーとも透明になります。
- **選択範囲でトリミング**: アクティブな選択範囲(または浮動片)に合わせてキャンバスを切り抜きます。選択がない場合は何もしません。
- **選択範囲を新規タブに複製**: 選択(または浮動片)が無ければ無効です。浮動片がある場合はそのピクセルを
  そのまま新規タブの唯一のレイヤーにし、静的な選択だけの場合は選択の bbox で全レイヤーを切り抜いて
  レイヤー構成(名前・表示・不透明度・重ね順・アクティブレイヤー)を保ったまま複製します。
  新規タブは元タブの直後に挿入されアクティブになり、元タブは一切変更されません(非破壊)。

「画像」グループの色調補正アイコン(明るさ・コントラスト/色相・彩度・明度/階調の反転/グレースケール化)は**アクティブレイヤーのみ**が対象(全レイヤーではありません)で、選択範囲があればその中だけに効きます。1 回の適用が 1 つの undo 単位になります。

|---|---|
| 明るさ・コントラスト…(ライブプレビュー) | メニューのみ |
| 色相・彩度・明度…(ライブプレビュー) | `Ctrl+U` |

## ビルド方法

前提: Rust stable(x86_64-pc-windows-msvc)がインストールされていること。

```powershell
# 開発ビルド
cargo build

# テスト
cargo test

# 配布用リリースビルド(単一 exe を target/release/darask-paint.exe に生成)
cargo build --release
```

依存 crate は `eframe` / `image` / `rfd` / `arboard` / `ab_glyph`(テキストツールのラスタライズ用)の
5つのみです(`Cargo.toml` 参照)。ビルド時専用の build-dependency として `winresource`
(exe への `assets/icon.ico` 埋め込み用、Windows 以外では `build.rs` が何もしません)を使っています。

### アイコンの再生成

`assets/icon.ico` はコードで描いた図形(`src/icon.rs`)から生成しています。デザインを変えたときは
以下で再生成してコミットしてください(exe 埋め込み・ウィンドウアイコンの両方が同じ関数を参照するため、
このファイルの再生成だけで見た目が揃います)。

```powershell
cargo run --example gen_icon
```

### 起動ベンチマーク(スモークテスト)

GUI を対話的に操作せずに起動時間を計測できます。環境変数 `DARASK_BENCH=1` を設定して実行すると、
2 フレーム目描画後にカレントディレクトリの `bench.txt` に起動ミリ秒を書き出して自動終了します。

```powershell
$env:DARASK_BENCH = '1'
& .\target\release\darask-paint.exe
Get-Content .\bench.txt
```

手元の実測値: **約 190〜220ms**(目標 300ms 以内)、exe サイズ **約 7.3MB**(目標 15MB 以下)。

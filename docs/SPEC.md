# Darask Paint — 仕様書 (v1)

## 0. 背景と最重要要件

Krita の起動が遅いことへの不満が出発点。**「超高速起動・軽量・シンプル」** なラスタ画像編集ソフトを作る。
迷ったら常に「起動速度・軽さ・シンプルさ」を優先して判断すること。

数値目標(リリースビルド、通常のデスクトップPC):

| 項目 | 目標 |
|---|---|
| 起動時間(プロセス起動 → 最初のフレーム描画) | **300ms 以内** |
| 配布物 | **単一 exe、15MB 以下**(ランタイム・インストーラ不要) |
| アイドル時 CPU 使用率 | **ほぼ 0%**(入力がない限り再描画しない) |
| 描画追従 | 4000×4000px 画像でブラシが 60fps で追従 |

## 1. 技術スタック

- Rust stable (x86_64-pc-windows-msvc)、edition 2021
- GUI: **eframe / egui**(glow レンダラ、デフォルト機能で可)
- 画像 I/O: **image** crate(`--no-default-features -F png,jpeg,bmp,gif,webp`)
- ファイルダイアログ: **rfd**
- クリップボード: **arboard**
- 依存はこの4つのみ。**安易に crate を追加しない**(コンパイル時間・バイナリサイズ・起動時間に直結)。

`main.rs` 先頭に `#![windows_subsystem = "windows"]`(コンソール窓を出さない)。

リリースプロファイル:

```toml
[profile.release]
opt-level = 3
lto = "thin"
codegen-units = 1
strip = "symbols"
panic = "abort"
```

## 2. 非目標

筆圧 / 芸術的フィルタ(ぼかし・シャープ等。基本的な色調補正は v4 §24 で解除)/ 印刷 / プラグイン / UI 多言語切替(**UI は日本語固定**)/ レイヤー付きネイティブ保存形式 / レイヤーのブレンドモード / フェザー(ぼかし)付き選択 / 磁性なげなわ / ブラシのカスタム形状。
これらを「ついで」に実装しないこと。
(段階的に解除してきた旧非目標: レイヤー=v2 §13、テキスト=v3 §19、非矩形選択・グラデーション・色調補正・設定の永続化=v4 §21〜§26)

├──────┬──────────────────────────────┬───────────────┤
│ ツール│ オプションバー: [サイズ][ツール固有] │ 右パネル       │
│ バー  ├──────────────────────────────┤  色(§14):     │
│ (縦、 │                              │   スウォッチ    │
│ アイコ│        キャンバスビュー        │   ホイール+三角 │
│ ン    │  (暗灰色の作業領域の中に        │   パレット      │
│ §15) │   画像を表示)                 │  レイヤー(§13) │
├──────┴──────────────────────────────┴───────────────┤

右パネルは固定幅 約210px。色スウォッチとカラーピッカーはオプションバーから右パネルへ移動(v2)。

- ウィンドウ: 初期 1280×800、最小 640×480、タイトル `{ファイル名}{*} - Darask Paint`(`*` は未保存変更あり。無題なら `無題`)。
- 起動時は **新規 1280×720・白背景** のドキュメントを自動作成(MS ペイント方式)。
- コマンドライン引数 `darask-paint.exe 画像.png` でそのファイルを開く(「プログラムから開く」対応)。
- 透明ピクセルの下には市松模様(チェッカーボード)を表示。
- キャンバス外の作業領域は暗灰色。

## 4. ツール(左縦ツールバー)

各ボタンはアイコン風の1文字(絵文字/記号)+ツールチップ(日本語名とショートカット)。

| ツール | キー | 挙動 |
|---|---|---|
| ペン | B | 丸筆・ハードエッジ(デフォルト)。オプションでアンチエイリアス ON/OFF。左ドラッグ=プライマリ色、**右ドラッグ=セカンダリ色**。ポインタイベント間は線分補間して途切れない線にする |
| 消しゴム | E | 丸筆ハードエッジで **透明(alpha=0)** にする |
| 直線 | L | ドラッグでプレビュー、離して確定。Shift で 45° 単位に拘束。太さ=ブラシサイズ、端は丸 |
| 矩形 | R | ドラッグでプレビュー→確定。モード: 枠線のみ / 塗りつぶし / 両方(枠=プライマリ、中=セカンダリ)。Shift で正方形 |
| 楕円 | C | 矩形と同様。Shift で正円 |
| 塗りつぶし | F | スキャンラインflood fill。許容値スライダー 0–255(デフォルト 0)。連結領域のみ |
| スポイト | I | 左クリック=プライマリに、右クリック=セカンダリに取得。**描画系ツール使用中も Alt+クリックで一時スポイト** |
| 選択 | M | §6 参照 |
| 手のひら | H | ドラッグでパン。**どのツール中でも Space 押下中ドラッグ・中ボタンドラッグでパン可** |

- ブラシサイズ: 1–64px、オプションバーのスライダー、`[` / `]` で減増。
- 単一キーのツールショートカットは、テキスト入力中(`ctx.wants_keyboard_input()`)は無効にすること。

## 5. 色

- オプションバーに プライマリ/セカンダリ の2スウォッチ(重ね表示)。クリックで egui 標準のカラーピッカーがポップアップ(`color_edit_button_srgba`)。
- `X` で入れ替え。
- 最近使った色 8 個を横並び表示(クリックでプライマリに)。描画確定時に使用色を先頭に追加(重複は先頭へ移動)。
- 色は RGBA(アルファ含む)。

## 6. 選択(フローティング方式)

- ドラッグで矩形選択(破線表示)。
- 選択内部をドラッグ → その領域を**浮動化**(元位置は透明で埋める)して移動。以後のドラッグは浮動片の移動。
- `Delete`: 選択領域(または浮動片)を消去(透明化)。
- `Ctrl+C` / `Ctrl+X`: コピー / 切り取り(切り取りは透明で埋める)。
- `Ctrl+V`: クリップボード画像をビュー中央に浮動片として貼り付け。**ただしドキュメントが完全に未編集・未保存(起動直後の白紙)の場合は、ドキュメント自体を貼り付け画像のサイズに置き換える**(スクリーンショット→保存が最短になるように)。
- 確定: `Enter`、選択外クリック、ツール切替 → 浮動片をその位置に合成し 1 回の undo 単位にする。`Esc` = その場で確定して選択解除。
- `Ctrl+A`: 全選択。
- 浮動片はキャンバス外にはみ出してよい(確定時にクリップ)。

## 7. メニュー

- **ファイル**: 新規 (Ctrl+N) / 開く (Ctrl+O) / 上書き保存 (Ctrl+S) / 名前を付けて保存 (Ctrl+Shift+S) / 終了 (Alt+F4)
- **編集**: 元に戻す (Ctrl+Z) / やり直し (Ctrl+Y, Ctrl+Shift+Z) / 切り取り / コピー / 貼り付け / 削除 / すべて選択 / 選択解除 (Ctrl+D)
- **画像**: 画像サイズ変更… / キャンバスサイズ変更… / 選択範囲でトリミング / 左右反転 / 上下反転 / 右に90°回転 / 左に90°回転
- **表示**: 拡大 (Ctrl++) / 縮小 (Ctrl+-) / 100% (Ctrl+1) / ウィンドウに合わせる (Ctrl+0)

ダイアログ仕様:
- **新規**: 幅・高さ(1–8192、デフォルト 1280×720)、背景 = 白 / 透明。
- **画像サイズ変更**: 幅・高さ(縦横比固定チェック、デフォルトON)、補間 = バイリニア / ニアレスト。
- **キャンバスサイズ変更**: 幅・高さ。既存画像は左上基準で配置、拡張部分は透明。
- ダイアログは egui のモーダル(`egui::Modal` が使えるバージョンならそれを使用)。

## 8. ファイル I/O・クリップボード

- 読み込み: PNG / JPEG / BMP / GIF(先頭フレーム)/ WebP。内部表現は常に RGBA8。
- 保存: PNG(デフォルト)/ JPEG / BMP。拡張子で判定、不明な拡張子なら `.png` を付ける。
- JPEG 保存時は品質ダイアログ(1–100、デフォルト 90)。アルファは白に合成してから保存。
- ドラッグ&ドロップでファイルを開く。
- 未保存変更ガード: 閉じる / 新規 / 開く の前に「保存しますか?」モーダル(保存 / 破棄 / キャンセル)。ウィンドウの閉じる要求は `close_requested` を検知して `ViewportCommand::CancelClose` で止めてから確認。
- I/O エラーで**絶対にパニックしない**。ステータスバーにトースト(約4秒表示)でエラー内容を出す。

## 9. アンドゥ / リドゥ

- `Ctrl+Z` / `Ctrl+Y`。
- 履歴メモリ上限 **256MB**、超えたら古いものから破棄。ただし直近 10 件は必ず保持。
- 1 ストローク・1 図形・1 塗りつぶし・1 貼り付け確定 = 1 undo 単位。
- サイズ変更・回転・トリミング等は全画像スナップショットの undo 単位。

## 10. ズーム / パン

- `Ctrl+ホイール`: カーソル位置を中心にズーム(段階 ×1.25)。範囲 5%–3200%。
- ホイール: 縦パン、`Shift+ホイール`: 横パン。
- 100% 以上はニアレスト補間(ピクセルがくっきり見える)、100% 未満はリニア。
- 100% 表示は物理ピクセル 1:1(DPI スケールに負けない)。
- パンは画像が完全に画面外へ消えないようクランプ(最低 32px は見える)。

## 11. 起動ベンチマークモード(必須機能)

環境変数 `DARASK_BENCH=1` で起動した場合:
- `main()` 冒頭で `Instant::now()` を記録。
- 2 フレーム目の `update()` 終了時に、経過ミリ秒をカレントディレクトリの `bench.txt` に書き出して `std::process::exit(0)`。
- これが起動時間目標(300ms)の検証手段であり、GUI を対話せずにスモークテストする手段でもある。

## 12. 品質基準

- `cargo build` は **エラー 0・警告 0**。`cargo fmt` 適用済み。
- `unwrap()` / `expect()` を I/O・ユーザー入力経路で使わない(プログラミング上の不変条件のみ可)。
- 純粋ロジック(ラスタ演算、座標変換、履歴、flood fill)にはユニットテスト必須。`cargo test` 常時グリーン。
- UI 文字列はすべて日本語。フォントは Windows システムフォント(游ゴシック等)を実行時ロード(ARCHITECTURE.md 参照)。

# v2 拡張仕様

v1 の全要件(起動 300ms・単一 exe 15MB・アイドル CPU 0%・警告 0 等)は v2 でも維持する。

## 13. レイヤー(最小実装)

- ドキュメントは 1 枚以上のレイヤーを持つ(下から上に合成、通常合成のみ、ブレンドモードなし)。上限 64 枚。
- 各レイヤー: 名前 / 表示・非表示 / 不透明度 0–100%。
- 描画系ツール(ペン・消しゴム・図形・塗りつぶし)、選択・浮動片、クリップボード操作は**アクティブレイヤーのみ**に作用する。
- スポイトは**合成結果**から色を取る。塗りつぶしの連結判定はアクティブレイヤーのピクセルで行う。
- 消しゴムはアクティブレイヤーを透明化(全レイヤー透明の部分は市松模様が見える)。
- 画像メニューの操作(サイズ変更/キャンバスサイズ/トリミング/反転/回転)は**全レイヤー**に適用。
- 新規作成・ファイルを開いた直後は「背景」レイヤー 1 枚。新規レイヤーは透明で名前は「レイヤー N」。
- 保存は常に**統合(可視レイヤーの合成)**して書き出す(PNG は合成アルファ保持、JPEG/BMP は白に合成)。複数レイヤーがある状態で保存したら「レイヤーは統合して保存されました」とトースト通知。レイヤー構成を保存できるネイティブ形式は非目標(§2)。
- 右パネルのレイヤーセクション:
  - 一覧(最上位レイヤーが一番上に表示)。行 = 表示チェック + 名前。クリックでアクティブ化(ハイライト)、ダブルクリックで名前変更。
  - ボタン: 新規 / 複製 / 削除 / 上へ / 下へ / 下と結合。レイヤーが 1 枚のときは削除・結合は無効。
  - アクティブレイヤーの不透明度スライダー。
- メニュー「レイヤー」: 新規レイヤー (Ctrl+Shift+N) / レイヤーを複製 / レイヤーを削除 / 上へ移動 / 下へ移動 / 下のレイヤーと結合 (Ctrl+E) / 画像の統合 (Ctrl+Shift+E)。
- アンドゥ: 描画パッチはレイヤー番号付き。レイヤーの追加/複製/削除/移動/結合/統合も 1 undo 単位。**表示切替と不透明度変更は履歴に積まない**(仕様として明記)。
- レイヤー操作・アンドゥは浮動片やストローク進行中にはツール切替と同じ扱い(先に確定してから実行)。

## 14. カラーパネル(色相リング + SV 三角形 + パレット)

右パネル上段。egui 標準ポップアップピッカーは廃止し、以下を常時表示する:

1. **スウォッチ**: プライマリ/セカンダリの重ね表示 + 入替ボタン(X キーと同じ)。編集対象は常にプライマリ。
2. **色相リング + SV 三角形**(直径約 170px):
   - 外周リング(ドーナツ)で色相を選択。0°=赤を 12 時方向、時計回り。リング上に現在色相のマーカー。
   - 内接する**固定向きの正三角形**(頂点: 上=純色相 S=1,V=1 / 左下=黒 V=0 / 右下=白 S=0,V=1)で彩度・明度を選択。三角形内に現在 S/V のマーカー。
   - リング内ドラッグ=色相のみ、三角形内ドラッグ=S/V のみ変更(ドラッグ開始領域で確定し、外れてもクランプ)。
3. **アルファスライダー**(0–255、市松模様の下地付き)と **HEX 入力欄**(#RRGGBB / #RRGGBBAA、確定で反映)。
4. **パレット**: 定義済み 24 色(2 段 × 12。1 段目: 黒 #000000, 灰 #7F7F7F, 白 #FFFFFF, 赤 #E53935, 橙 #FB8C00, 黄 #FDD835, 緑 #43A047, 水 #00ACC1, 青 #1E88E5, 紫 #8E24AA, 桃 #EC407A, 茶 #6D4C41。2 段目は各色の明るいパステル調)。クリック=プライマリ、右クリック=セカンダリ。
   - 「＋」でプライマリ色をユーザーパレットに追加、ユーザー色は右クリックメニューで削除。永続化はしない(§2)。
5. **最近使った色 8 個**(v1 のものをオプションバーからここへ移動)。

## 15. ツールアイコン

- 左ツールバーの漢字 1 文字ボタンを、**egui Painter によるベクター描画アイコン**(約 20×20、線幅 ~1.5pt)に置き換える。画像アセット・絵文字フォントは使わない(サイズ・起動時間・DPI・テーマ追従のため)。
- 9 ツール: ペン=鉛筆 / 消しゴム=傾いた角丸矩形 / 直線=斜め線 / 矩形=矩形枠 / 楕円=楕円枠 / 塗りつぶし=バケツ+滴 / スポイト=斜めのスポイト / 選択=破線矩形 / 手のひら=手(または十字矢印)。
- 色は egui のテキスト色(選択中はアクセント色)に追従。ツールチップ(日本語名+ショートカット)は維持。
- 一目で区別でき、20×20 で潰れないこと。

## 16. 選択・浮動片のスケールハンドル(※ Esc の挙動は v3 §18 で「キャンセル」に変更)

- 選択矩形および浮動片の外周に **8 個のハンドル**(四隅+各辺中央、約 7pt 角)を表示。
- ハンドルをドラッグすると拡縮(四隅=両軸、辺=単軸)。**Shift で縦横比固定**。アンカーは反対側の隅/辺。
- 未浮動の選択でハンドルを掴んだ場合は、内部ドラッグと同様にまず浮動化してから拡縮する。
- 拡縮は**浮動化時の元ピクセルから毎回バイリニアで再サンプリング**する(累積劣化させない)。最小 1px、最大 8192px。
- ハンドルホバー時はリサイズカーソルを表示。移動・確定・Esc 等の挙動(§6)は不変。
- 従来どおりメニューにも 画像サイズ変更 / キャンバスサイズ変更 / 選択範囲でトリミング がある(こちらは全レイヤー対象)。

## 17. ブラシエンジン(硬さ・不透明度)

ブラシ(旧ペン)・消しゴムを共通のストロークエンジンに刷新する:

- **硬さ** 0–100%(デフォルト 100)。半径 r に対し r×硬さ までカバレッジ 1、そこから外周 r までなめらかに減衰(smoothstep)。Shift+[ / Shift+] で ±10。
- **不透明度** 1–100%(デフォルト 100)。**Photoshop の opacity 意味論**: 1 ストローク内で何度重ねてもそのストロークの被覆は不透明度上限を超えない(カバレッジは max 合成)。ストロークをまたげば重なる。
- **鉛筆モード**(チェックボックス、デフォルト OFF): アンチエイリアスなしの2値スタンプ(ピクセルアート用)。硬さ無視、不透明度は有効。旧「アンチエイリアス」チェックは廃止(ブラシは常時 AA)。
- 消しゴムも同エンジン(不透明度=**強さ**として表示。カバレッジ×強さぶんアルファを減らす)。
- **Shift+クリック**: 直前のストローク終点から Shift+クリック地点まで直線をブラシで引く(PS 準拠)。
- **数字キー 1–9, 0**: ブラシ系ツールの不透明度を 10%〜90%, 100% に設定(テキスト入力中は無効)。
- **ブラシカーソル**: キャンバス上ではブラシ半径の円アウトライン(白+黒の二重線で背景を問わず視認可)を表示し、OS カーソルは非表示。画面上の円が 3px 未満になる場合は十字カーソルにフォールバック。

## 18. 移動ツール・ズームツール・自由変形・Esc の挙動変更

- **移動ツール (V)**: ドラッグで**アクティブレイヤー全体**を移動。選択範囲があればその範囲だけを移動(選択ツールの内部ドラッグと同じ浮動化)。実装はどちらも「自動浮動化→移動→確定」で統一(§16 のハンドルも表示され、そのまま拡縮可能)。
- **ズームツール (Z)**: クリック=クリック位置を中心に 1 段階拡大、Alt+クリック=縮小。カーソルは虫眼鏡。
- **Ctrl+T(自由変形)**: 選択範囲があれば浮動化してハンドル表示。なければ全選択→アクティブレイヤーを浮動化してハンドル表示。以降は §16 と同じ操作。
- **Esc の挙動変更(v1 §6 を上書き)**: Esc は**キャンセル** — 浮動片を破棄して元の位置・内容に完全復元し(切り出し元も戻す)、選択を解除する。履歴には何も積まない。確定は従来どおり Enter / 選択外クリック / ツール切替。テキスト編集中の Esc も入力破棄。

## 19. テキストツール (T)

- クリック位置にインラインのテキスト入力ボックス(egui TextEdit、複数行、IME 対応)を表示。入力中は全ツール系ショートカット無効。
- オプションバー: フォントサイズ 8–144px(デフォルト 24)。色はプライマリ。フォントは UI と同じシステム日本語フォント 1 種のみ(太字・斜体・フォント選択は非目標)。
- 確定(Ctrl+Enter または ボックス外クリック)でアンチエイリアス付きラスタライズし、**浮動片として配置**(移動・ハンドル拡縮可、Enter 等で通常確定=1 undo 単位)。Esc は入力破棄。
- 空文字列の確定は何もしない。クリック位置=テキストボックスの左上。行送りはフォントメトリクス準拠(約 1.3×サイズ)。
- 実装のため **ab_glyph を第 5 の依存として許可**する(`=0.2.32` に `=` 固定。当初は「egui が既に内部使用している」想定だったが、egui/epaint 0.35 は内部で ab_glyph を使っておらず(skrifa/harfrust 系に移行済み、`cargo tree -i ab_glyph` で確認可能)、darask-paint 単独の追加依存になっている。そのため「egui と同じバージョンに揃える」という制約自体は成立しないが、再現性のためのバージョン固定(`=`)は引き続き必須。ab_glyph 単体の追加コスト(`ab_glyph_rasterizer`/`owned_ttf_parser`/`ttf-parser` を含め計4クレート)はコンパイル時間・バイナリサイズへの影響が「実質ゼロ」というほどではないが小さい)。

# v4 拡張仕様(選択の一般化・グラデーション・色調補正・永続化・性能・CI/CD)

v1〜v3 の全要件(単一 exe 15MB・アイドル CPU 0%・警告 0 等)は v4 でも維持する。起動時間の目標は §28 で更新する。

## 21. マスク選択(選択機能の一般化)

矩形限定だった選択を**ビットマスク選択**に一般化する。v4 の全選択ツール・全編集操作の土台。

- 選択 = `bbox + バイナリマスク`(画素単位の 選択/非選択。フェザーは非目標)。
- 選択枠の表示は**マスク境界線**(選択画素と非選択画素の境界)を実線+破線で描く(アニメーションなし、従来どおり)。
- 浮動化・移動・スケールハンドル・コピー/切り取り/貼り付け/Delete・Ctrl+T は**マスク形状のまま**動作する(浮動片は非矩形になれる。スケール時はピクセルとマスクを同時に再サンプル)。
- **描画のクリップ**: 選択がある間、ブラシ/消しゴム/鉛筆/図形/塗りつぶし/グラデーション/色調補正は**選択範囲内にのみ**効く(カバレッジ×選択マスクで自然に合成)。塗りつぶしの連結探索も選択内に制限。
- 「選択範囲でトリミング」は bbox でトリミング。
- 既存の矩形選択は「マスクが全 1 の矩形」として同一コードパスに載せ替える(挙動不変)。

## 22. 選択ツールの拡充

| ツール | キー | 挙動 |
|---|---|---|
| 矩形選択 / 楕円選択 | M / **Shift+M で巡回** | 従来の矩形に加え楕円マスク。Shift ドラッグで正方形/正円 |
| なげなわ | **L**(**Shift+L** で自由↔多角形) | 自由: ドラッグの軌跡を閉じてマスク化。多角形: クリックで頂点追加、ダブルクリック/Enter/始点クリックで閉じる、Esc で中止 |
| 自動選択 | **W** | クリック画素から許容値(0–255、オプションバー)の連結領域をマスク選択(flood fill と同じ判定、アクティブレイヤー基準) |

- いずれも確定済み選択がある状態での新規選択は**置き換え**(加算/減算モードは非目標。ただし実装を将来拡張できる構造にする)。

## 23. グラデーションツール

- キー: **G**(**Shift+G** で 塗りつぶし↔グラデーション巡回)。
- ドラッグで始点→終点。ドラッグ中はプレビュー線+ライブプレビュー、離して確定(1 undo 単位)。
- 種類: **線形 / 円形**(オプションバー)。色: **プライマリ→セカンダリ / プライマリ→透明**(オプションバー)。
- 始点前/終点後はクランプ(端色で埋める)。選択範囲があればその中だけ。アクティブレイヤー対象。AA/ディザは無し(将来課題)。

## 24. 色調補正(画像メニュー > 色調補正)

すべて**アクティブレイヤー**対象、**選択範囲があればその中だけ**、各適用が 1 undo 単位。

- **明るさ・コントラスト…** — 各 −100〜+100 のスライダー、**ライブプレビュー付き**モーダル(OK/キャンセル。キャンセルで完全復元)。
- **色相・彩度・明度… (Ctrl+U)** — 色相 −180〜+180、彩度/明度 −100〜+100。同上のモーダル。
- **階調の反転 (Ctrl+I)** — 即時(RGB反転、アルファ不変)。
- **グレースケール化 (Ctrl+Shift+U)** — 即時(Rec.709 輝度)。
- ライブプレビューは不透明度スライダーと同じ「値が変わったフレームだけ再適用」方式。モーダル中の再適用は元画素スナップショットから毎回計算(累積適用しない)。

## 25. ブラシスムージング・ピクセルグリッド

- **スムージング(手ブレ補正)**: ブラシ/消しゴム/鉛筆のオプションに 0–100%。入力座標の指数移動平均(0=補正なし)。ストローク終端で最終点まで追いつかせる。Shift+クリック直線連結には適用しない。
- **ピクセルグリッド**: 表示メニューにトグル。ズーム 800% 以上のとき画像ピクセル境界に薄いグリッド線を描く(デフォルト ON)。

## 26. 設定の永続化・最近使ったファイル・バージョン情報

軽量な独自形式(`キー\t値` の UTF-8 テキスト、外部 crate 禁止・手書きパーサ+テスト)で `%APPDATA%\darask-paint\settings.txt` に保存する。**読み込みは起動時 1 回、書き込みは終了時と最近使ったファイル更新時のみ。破損・欠損時は黙って既定値**(起動を絶対に妨げない。§28 の起動目標に算入)。

永続化する項目(これ以外は追加しない): ウィンドウ寸法・最大化状態 / 最近使ったファイル(最大 8)/ ブラシ設定(サイズ・硬さ・不透明度・鉛筆・スムージング)/ ツールオプション(許容値・図形モード・グラデーション設定)/ プライマリ・セカンダリ色 / ユーザーパレット / 最後に使ったツール / ピクセルグリッド表示。

- **ファイル > 最近使ったファイル** サブメニュー(存在しないパスは選択時に一覧から除去してトースト)。
- **ヘルプ > バージョン情報**: バージョン(`CARGO_PKG_VERSION`)・リポジトリ URL を表示する小モーダル(メニューバーに「ヘルプ」追加)。

## 28. パフォーマンス目標(v4 で強化)

計測してから最適化する(推測での最適化禁止)。リリースビルドでの受け入れ基準:

| 項目 | 目標 |
|---|---|
| 起動(→初フレーム) | **中央値 160ms 以下**を目指し、300ms 以内は必須。`DARASK_BENCH=1` の bench.txt にフェーズ内訳(設定読込/フォント/ウィンドウ作成/初フレーム)を出力する |
| ブラシ追従 | 4000×4000・ブラシ64px の高速ドラッグでフレーム 16ms 以下(ドラッグを横断する巨大 dirty 矩形を作らない — セグメント単位の dirty 処理) |
| 履歴メモリ | ストロークの Patch を**タイル単位のサブパッチ集合**にし、離れた 2 点を触っても bbox 合併で面積爆発しない |
| 浮動片ハンドル | ドラッグ中の再サンプルはフレームあたり最大 1 回 |
| Esc キャンセル | 全レイヤー浮動化のキャンセル復元をタイル一括コピーで 50ms 以下(4000×4000) |
| ホットループ | composite / blend / flood fill を行スライス処理にし、画素単位の境界チェック呼び出しを排除 |

- `DARASK_DEBUG=1` のときのみ、フレーム時間のデバッグオーバーレイを表示してよい(通常起動には一切のコストを載せない)。

## 29. CI/CD・リリース

- **CI**(`.github/workflows/ci.yml`、windows-latest、main への push と PR):
  `cargo fmt --check` → `cargo clippy --all-targets -- -D warnings` → `cargo build --release` → `cargo test --release`。Rust キャッシュ利用。
  ベンチ(GUI 起動)はランナーがヘッドレスの可能性があるため `continue-on-error: true` の別ジョブ。
- **リリース**(`.github/workflows/release.yml`、`v*` タグ push 時):
  リリースビルド → exe を `darask-paint-v{X.Y.Z}-windows-x64.zip` に固めて GitHub Release を自動作成(自動生成ノート付き)。
- **clippy 警告 0** を新たな品質基準に追加する(既存コードの全 lint を解消してから CI を入れる)。
- **exe アイコン**: `assets/icon.ico`(自前生成でよい。生成スクリプトは `examples/gen_icon.rs`、image crate の ico 機能で出力)+ `build.rs` + build-dependency **winresource**(build-dependency はランタイム依存ではないため許可。これ以外の build-dependency は追加しない)。
- バージョンを **0.4.0** に上げる。以後、リリースはタグ push で行う。

# v5 拡張仕様(複数ドキュメント・タブ)

v1〜v4 の全要件(起動 300ms・単一 exe 15MB・アイドル CPU 0%・警告 0・clippy 0 等)は v5 でも維持する。

**読み替え規則**: v1〜v4 で「ドキュメント」「doc」「画像」と書かれている箇所は、v5 以降は**アクティブタブのドキュメント**を指す。ツール・色・ブラシ設定・パレット・最近使ったファイル・ウィンドウ状態は引き続きアプリ全体で共有(タブごとには持たない)。選択・浮動片・アンドゥ履歴・ズーム/パン・ファイルパス・未保存フラグは**タブごと**に独立する。

## 30. タブ(複数ドキュメント)

- メニューバーの直下、ツールバー/オプションバーの上に**タブバー**を新設(横1列、水平スクロール、多数タブでも折り返さない)。
- 各タブ: ファイル名(無題なら「無題」「無題2」「無題3」…と連番)+ 未保存インジケータ(`*`)+ 閉じるボタン(×)。クリックでアクティブ化、**中クリックでも閉じる**。
- **常に 1 タブ以上を維持する**: 最後の 1 タブを閉じようとした場合、そのタブを閉じるのではなく「新規」と同じ扱い(未保存ガードを通してから内容を白紙に戻す)にする。タブが 0 枚になる状態は作らない。
- タブ数の上限は **24**。超えて新規タブを作ろうとしたら作成せずトースト通知(SPEC §13 のレイヤー上限と同じ流儀)。
- **Ctrl+N(新規)** / **Ctrl+O(開く)** は、従来の「アクティブタブの内容を置き換える」から「**新規タブを追加してそこに開く**」に変更する(v1 §7・§8 を上書き)。開こうとしたファイルが既に開いているタブがあれば(パスを正規化して比較)、新規タブを作らずそのタブへ切り替える。ドラッグ&ドロップ・最近使ったファイルからのオープンも同じ規則に従う。複数ファイルを同時ドロップした場合は複数タブを開く。
- **Ctrl+W**: アクティブタブを閉じる(未保存なら確認ダイアログ)。**Ctrl+Tab** / **Ctrl+Shift+Tab**: 次のタブ / 前のタブへ切り替え(端では反対側へ循環)。
- ウィンドウの閉じる操作(×ボタン・Alt+F4・Ctrl+Q 相当)は、未保存のタブがあれば**タブごとに順番に**確認ダイアログを出す(いずれかでキャンセルすれば閉じる処理全体を中止)。
- ウィンドウタイトルは引き続き `{アクティブタブのファイル名}{*} - Darask Paint` を表示する。
- タブの並べ替え(ドラッグでの順序変更)は非目標(v5 では実装しない、シンプルさ優先)。
- セッション復元(前回開いていたタブ一覧の記憶)は非目標。起動時は常にタブ 1 枚(コマンドライン引数があればそれを開いた状態、無ければ白紙の新規)。

## 31. 選択範囲を新規タブに複製

- 画像メニューに「**選択範囲を新規タブに複製**」を追加(「選択範囲でトリミング」の直後)。**選択または浮動片がアクティブなときのみ有効**(無ければグレーアウト、他の選択依存メニュー項目と同じ扱い)。
- 挙動:
  - **浮動片がある場合**: その浮動片のピクセル(mask込み)をそのまま新規タブの唯一のレイヤーにする。
  - **静的な選択のみの場合**: 選択マスクの bbox で、**全レイヤー**をマスク外transparentで切り出し、レイヤー構成(名前・表示・不透明度・重ね順・アクティブレイヤー)を保ったまま新規タブに複製する(「選択範囲でトリミング」の全レイヤー方針を踏襲)。
- 元のタブは一切変更しない(非破壊)。新規タブは元タブの**直後**に挿入されアクティブになる。新規タブの未保存フラグは **true**(ファイルに存在しない新しい内容のため)、パスは無し(「無題」系の命名)、アンドゥ履歴は空から始まる。

## 32. v5 キーマップ追加(§20 に追記)

| キー | 動作 |
|---|---|
| Ctrl+W | タブを閉じる |
| Ctrl+Tab | 次のタブ |
| Ctrl+Shift+Tab | 前のタブ |
| 中クリック(タブ上) | そのタブを閉じる |

キーマップは引き続き keymap.rs の単一テーブルが正。

- 対象は以下の全項目(§7 ファイル/編集/画像/レイヤー/表示、§13 レイヤー、§19 テキスト由来の自由変形、§26 バージョン情報、v4/v5 の追加項目すべてを含む):

  **編集**: 元に戻す / やり直し / 切り取り / コピー / 貼り付け / 削除 / すべて選択 / 選択解除 / 自由変形。
  **画像**: 画像サイズ変更 / キャンバスサイズ変更 / 選択範囲でトリミング / 選択範囲を新規タブに複製 / 左右反転 / 上下反転 / 右に90°回転 / 左に90°回転 / 明るさ・コントラスト / 色相・彩度・明度 / 階調の反転 / グレースケール化。
  **レイヤー**: 新規レイヤー / レイヤーを複製 / レイヤーを削除 / 上へ移動 / 下へ移動 / 下と結合 / 画像の統合。
  **表示**: 拡大 / 縮小 / 100% / ウィンドウに合わせる / **ピクセルグリッド表示**(トグル、ON 時は選択中のツールボタンと同じハイライト表示)。
  **その他**: バージョン情報 / **設定**(新規、§34 参照)。
- 各アイコンの押下時の実際の動作(ダイアログを開く・即実行する・確認を要する等)は v1〜v5 の既存仕様から一切変更しない。**変わるのは見た目と配置だけ**(ドロップダウン → 常時表示アイコン)。
- レイアウト順は メニューバー(本節)→ タブバー(v5 §30)→ オプションバー → キャンバス/右パネル → ステータスバー。

## 34. 設定(環境設定)ダイアログ

- **Ctrl+K**(Photoshop 準拠)、または新設のアイコンボタンで開くモーダルダイアログ。
- 最小構成として **「元に戻す履歴の保持数」**(1–500、デフォルト **50**)の数値入力(スライダー or DragValue)のみを持つ。OK で即座に適用+設定ファイルへ保存、キャンセルで変更を破棄。
- 値を下げて即座に既存の保持数を下回った場合、開いている**全タブ**の履歴をその場で新しい上限まで切り詰める(§35 のメモリ会計と整合させる)。
- この値は SPEC §26 の永続化対象に追加する(`%APPDATA%\darask-paint\settings.txt`)。

## 35. アンドゥ履歴パネル

- 右パネルに「色」「レイヤー」に続く 3 番目のセクション「**履歴**」を追加(アクティブタブの履歴を表示。タブ切替で表示内容も切り替わる)。
- 一覧はスクロール可能なリストで、**上から下へ時系列(先頭 = 初期状態、末尾 = 最新)**。各行は操作の日本語ラベル 1 つ(例: 「ブラシ」「塗りつぶし」「画像サイズ変更」「レイヤーを追加」)。先頭には常に仮想的な「(初期状態)」行がある。
- **現在位置をハイライト**する。現在位置より後ろ(やり直し可能な範囲)の行は淡色表示にする。
- **行をクリックすると、その時点まで一気に戻る/進む**(複数手順のジャンプ)。ジャンプ後に新しい操作を行うと、淡色表示だった「やり直し」範囲は通常どおり破棄される(SPEC §9 の「新規 push で redo クリア」と同じ挙動)。
- ジャンプは既存の「元に戻す/やり直し」と同じ安全規則に従う: 進行中のストローク・浮動片があれば**先に確定してからジャンプする**(SPEC §13 最終項と同じ扱い。ジャンプだけを特別扱いしない)。
- 履歴メモリ上限(SPEC §9 の 256MB、および §34 の保持数上限)によって古いエントリが実際に破棄されている場合、リスト先頭に「(これ以前の履歴は破棄されました)」という灰色の注記行を表示する。
- 「元に戻す/やり直し」のツールバーアイコン(§33)は従来どおり**単発ステップ**の操作のまま(複数ステップのジャンプは履歴パネルのクリックでのみ行う)。
